## 新增需求

### 需求:数据库连接失败时应用必须优雅降级
当数据库不可用时，应用不应崩溃，而应显示基本功能。

#### 场景:数据库连接失败
- **当** 数据库连接失败
- **那么** 首页仍能正常显示
- **那么** 用户看到未登录状态的导航

#### 场景:认证失败
- **当** auth() 调用失败
- **那么** Header 组件仍能渲染
- **那么** 显示登录按钮而非用户下拉菜单

### 需求:环境变量必须正确配置
应用必须验证必需的环境变量是否已配置。

#### 场景:缺少数据库连接字符串
- **当** POSTGRES_URL 未配置
- **那么** 应用启动时打印明确的错误信息

#### 场景:环境变量已配置
- **当** 所有必需环境变量已配置
- **那么** 应用正常运行
