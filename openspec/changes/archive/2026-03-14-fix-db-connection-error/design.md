## 上下文

首页加载时会调用 Header 组件，Header 调用 `auth()` 进行用户认证，`auth()` 使用 DrizzleAdapter 连接数据库。当数据库连接失败时，整个页面崩溃返回 500 错误。

## 目标 / 非目标

**目标：**
- 在数据库连接失败时，应用仍能显示基本页面
- 提供清晰的错误提示，帮助开发者诊断问题
- 确保用户看到友好的错误信息而非 500 错误

**非目标：**
- 重新设计认证流程
- 添加数据库重连逻辑

## 决策

### 1. 数据库连接添加错误处理
- **理由**: 当前数据库连接直接抛出异常，没有错误处理
- **实现**: 在 `src/lib/db/index.ts` 中添加 try-catch，连接失败时返回 null 或抛出明确错误

### 2. Header 组件优雅降级
- **理由**: 即使数据库不可用，页面仍应显示
- **实现**: 用 try-catch 包裹 auth() 调用，失败时显示未登录状态

### 3. 添加环境变量验证
- **理由**: 很多 500 错误是由于环境变量未配置导致的
- **实现**: 在应用启动时检查必需的环境变量

## 风险 / 权衡

- **数据库连接缓存** → 当前实现每次都创建新连接，可能导致连接池耗尽，但这是现有问题
- **性能影响** → 添加错误处理对性能影响极小
