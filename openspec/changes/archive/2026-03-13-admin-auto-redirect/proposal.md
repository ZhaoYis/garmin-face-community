## 为什么

当前系统需要管理员手动访问 `/admin` 路径才能进入管理后台。对于超级管理员（由环境变量 `ADMIN_EMAILS` 配置），应该在他们登录后自动重定向到管理后台，提升管理效率。

## 变更内容

- 添加 `ADMIN_EMAILS` 环境变量配置
- 在用户登录成功后检测其邮箱是否在超级管理员列表中
- 如果是超级管理员，自动重定向到 `/admin` 管理后台
- 更新 auth 回调逻辑实现自动重定向

## 功能 (Capabilities)

### 新增功能

- `admin-auto-redirect`: 超级管理员登录后自动重定向到管理后台

### 修改功能

（无现有功能需要修改）

## 影响

- `.env.example` - 添加 `ADMIN_EMAILS` 配置说明
- `src/auth.ts` - 添加登录后重定向逻辑
- 可能需要更新登录成功后的回调处理
