## 新增需求

### 需求:用户可以查看运动记录列表
系统必须向已登录用户展示其运动记录列表，支持分页和筛选。

#### 场景:列表展示
- **当** 用户访问运动记录页面
- **那么** 系统按时间倒序展示运动记录，每条记录显示类型、日期、距离、时长

#### 场景:分页加载
- **当** 运动记录超过 20 条
- **那么** 系统支持分页或无限滚动加载更多

#### 场景:类型筛选
- **当** 用户选择特定运动类型（跑步/骑行/游泳）
- **那么** 系统仅展示该类型的运动记录

### 需求:用户可以查看运动记录详情
系统必须展示单条运动记录的完整详情。

#### 场景:详情页展示
- **当** 用户点击某条运动记录
- **那么** 系统展示完整数据：活动名称、日期、开始时间、时长、距离、平均配速、平均心率、最大心率、累计爬升、卡路里

#### 场景:路线展示
- **当** 运动记录包含 polyline 数据
- **那么** 系统在详情页展示路线地图预览

### 需求:未绑定 Garmin 用户必须看到绑定提示
对于未绑定 Garmin 的用户，系统必须提示绑定账号以查看运动记录。

#### 场景:未绑定提示
- **当** 未绑定 Garmin 的用户访问运动记录页面
- **那么** 系统显示"绑定佳明账号以同步您的运动数据"的提示和绑定按钮
