## 上下文

当前 UserDropdown 组件已有头像显示逻辑：
- 有 `user.image` 时显示头像图片
- 无头像时显示带有 User 图标的圆形背景

需要优化默认头像的视觉效果，使其更加专业美观。

## 目标 / 非目标

**目标：**
- 创建美观的默认头像，使用用户名首字母显示
- 添加头像加载失败的 fallback 处理
- 确保 OAuth 登录时正确获取头像 URL

**非目标：**
- 不实现用户自定义上传头像功能
- 不修改用户数据模型

## 决策

### 默认头像设计

选择使用**用户名首字母**作为默认头像，理由：
- 个性化体验，每个用户看到不同的默认头像
- 实现简单，无需额外的图片资源
- 常见的行业实践（如 Google、GitHub 等）

**替代方案：**
- 使用固定的默认头像图片 - 缺乏个性化
- 使用 User 图标 - 当前方案，效果不够专业

### 头像组件设计

创建独立的 `Avatar` 组件，使用 shadcn/ui 的 Avatar 组件：
- 支持 `src`、`alt`、`fallback` 属性
- 内置图片加载失败处理
- 统一的样式和尺寸

## 风险 / 权衡

- **首字母可能不准确**：如果用户名为空或为邮箱，需要处理 fallback 逻辑 → 使用 "U" 作为最终 fallback
- **头像加载失败**：网络问题可能导致头像无法加载 → Avatar 组件内置 fallback 处理
