## 新增需求

### 需求:创作者必须能够上传表盘作品
角色为 creator 的用户必须能够上传自己的表盘作品。

#### 场景:上传表盘
- **当** 创作者填写表盘信息并上传文件
- **那么** 系统存储表盘文件，创建 watch_faces 记录，status 设为 pending

#### 场景:非创作者上传
- **当** 非创作者用户尝试访问上传页面
- **那么** 系统重定向到 403 页面

### 需求:创作者必须能够管理自己的作品
创作者必须能够查看和编辑自己上传的表盘。

#### 场景:作品列表
- **当** 创作者访问"我的作品"页面
- **那么** 系统显示该用户上传的所有表盘，包含审核状态

#### 场景:编辑作品
- **当** 创作者修改自己表盘的信息
- **那么** 系统更新表盘记录，审核中的作品重新进入待审核队列

#### 场景:删除作品
- **当** 创作者删除自己的表盘
- **那么** 系统从数据库和存储中删除该表盘

### 需求:系统必须验证表盘文件格式
上传的表盘文件必须是 Garmin 支持的格式。

#### 场景:格式验证
- **当** 用户上传非 .garmin 或 .prg 格式的文件
- **那么** 系统拒绝上传并提示格式错误

#### 场景:文件大小限制
- **当** 用户上传超过 10MB 的文件
- **那么** 系统拒绝上传并提示文件过大
