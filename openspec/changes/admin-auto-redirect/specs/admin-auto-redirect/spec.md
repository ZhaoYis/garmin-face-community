## 新增需求

### 需求:系统必须支持通过环境变量配置超级管理员邮箱
系统必须允许通过 `ADMIN_EMAILS` 环境变量配置超级管理员邮箱列表。

#### 场景:单个管理员邮箱
- **当** 环境变量 `ADMIN_EMAILS` 设置为 `admin@example.com`
- **那么** 系统识别该邮箱为超级管理员

#### 场景:多个管理员邮箱
- **当** 环境变量 `ADMIN_EMAILS` 设置为 `admin1@example.com,admin2@example.com`
- **那么** 系统识别所有列出的邮箱为超级管理员

#### 场景:未配置管理员邮箱
- **当** 环境变量 `ADMIN_EMAILS` 未设置或为空
- **那么** 系统不进行超级管理员自动重定向

### 需求:超级管理员登录后必须自动重定向到管理后台
当超级管理员成功登录后，系统必须自动重定向到 `/admin` 管理后台。

#### 场景:超级管理员登录成功
- **当** 超级管理员邮箱的用户成功登录
- **那么** 系统自动重定向到 `/admin` 页面

#### 场景:普通用户登录成功
- **当** 非超级管理员用户成功登录
- **那么** 系统重定向到首页或之前请求的页面

#### 场景:超级管理员访问其他页面后登录
- **当** 超级管理员尝试访问受保护页面后被重定向到登录页
- **那么** 登录成功后重定向到管理后台，而非之前请求的页面

### 需求:超级管理员必须自动获得管理员角色
当超级管理员邮箱的用户登录时，系统必须自动将其角色设置为 admin。

#### 场景:新超级管理员首次登录
- **当** 超级管理员邮箱的新用户首次登录
- **那么** 系统创建用户记录并将 role 设置为 admin

#### 场景:现有超级管理员登录
- **当** 已存在的超级管理员用户登录且角色不是 admin
- **那么** 系统将其 role 更新为 admin
