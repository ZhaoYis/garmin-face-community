## 新增需求

### 需求:用户可以切换主题
系统必须允许用户在 Light、Dark 和 System（跟随系统）三种模式之间切换。

#### 场景:切换到 Light 模式
- **当** 用户点击主题切换按钮并选择 "Light"
- **那么** 页面立即切换到浅色主题，背景变亮，文字变深

#### 场景:切换到 Dark 模式
- **当** 用户点击主题切换按钮并选择 "Dark"
- **那么** 页面立即切换到深色主题，背景变暗，文字变亮

#### 场景:切换到 System 模式
- **当** 用户点击主题切换按钮并选择 "System"
- **那么** 页面根据系统偏好自动选择 Light 或 Dark 模式

### 需求:主题选择必须持久化
用户选择的主题必须保存到 localStorage，下次访问时自动应用。

#### 场景:重新访问应用
- **当** 用户关闭浏览器后重新访问应用
- **那么** 应用使用上次选择的主题

### 需求:Header 必须显示主题切换按钮
页面顶部的 Header 必须包含主题切换入口。

#### 场景:显示主题切换按钮
- **当** 用户查看 Header
- **那么** 可以看到一个主题切换图标按钮

#### 场景:点击显示选项
- **当** 用户点击主题切换按钮
- **那么** 显示下拉菜单包含 Light、Dark、System 三个选项

### 需求:所有页面必须正确响应主题变化
切换主题后，所有 UI 元素必须正确显示对应主题的颜色。

#### 场景:深色模式显示正确
- **当** 切换到 Dark 模式
- **那么** 所有背景、文字、边框、卡片等 UI 元素使用深色主题颜色
