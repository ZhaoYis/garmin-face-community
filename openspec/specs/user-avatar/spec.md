## 新增需求

### 需求:用户头像必须正确显示
系统必须在 Header 的用户下拉菜单中显示用户头像。

#### 场景:用户有 OAuth 头像
- **当** 用户通过 Google 或 GitHub 登录且 OAuth 提供者返回头像 URL
- **那么** 系统显示 OAuth 提供的头像图片

#### 场景:用户无头像时显示默认头像
- **当** 用户没有头像 URL
- **那么** 系统显示用户名首字母作为默认头像

#### 场景:用户名为空时显示默认占位
- **当** 用户没有头像 URL 且用户名为空
- **那么** 系统显示 "U" 作为默认占位符

### 需求:头像加载失败时必须显示 fallback
系统必须在头像图片加载失败时显示备用内容。

#### 场景:头像图片加载失败
- **当** 用户头像 URL 存在但图片加载失败
- **那么** 系统显示用户名首字母作为 fallback

#### 场景:头像图片加载失败且无用户名
- **当** 用户头像图片加载失败且用户名为空
- **那么** 系统显示 "U" 作为 fallback

### 需求:头像组件必须支持深色模式
头像组件必须正确响应主题变化。

#### 场景:深色模式下显示正确
- **当** 切换到 Dark 模式
- **那么** 默认头像背景和文字颜色适配深色主题
