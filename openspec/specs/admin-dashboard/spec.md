## 新增需求

### 需求:管理员必须能够访问后台管理页面
系统必须提供管理员专属的后台管理入口。

#### 场景:访问后台
- **当** 角色为 admin 的用户访问 /admin 路径
- **那么** 系统显示后台管理页面

#### 场景:非管理员访问
- **当** 非管理员用户访问 /admin 路径
- **那么** 系统重定向到 403 禁止访问页面

### 需求:管理员必须能够管理用户
系统必须提供用户管理功能。

#### 场景:用户列表
- **当** 管理员访问用户管理页面
- **那么** 系统显示所有用户列表，包含用户名、邮箱、角色、状态

#### 场景:修改用户角色
- **当** 管理员修改某用户的角色
- **那么** 系统更新该用户的 role 字段

#### 场景:禁用用户
- **当** 管理员禁用某用户
- **那么** 系统将该用户的 status 设置为 disabled

### 需求:管理员必须能够管理海报模板
系统必须提供模板管理功能。

#### 场景:模板列表
- **当** 管理员访问模板管理页面
- **那么** 系统显示所有海报模板列表

#### 场景:新增模板
- **当** 管理员创建新模板
- **那么** 系统在数据库中创建新的模板记录

#### 场景:编辑模板
- **当** 管理员编辑模板信息
- **那么** 系统更新模板记录

### 需求:管理员必须能够审核表盘作品
创作者上传的表盘需要管理员审核。

#### 场景:待审核列表
- **当** 管理员访问表盘审核页面
- **那么** 系统显示所有待审核（status=pending）的表盘列表

#### 场景:审核通过
- **当** 管理员批准某表盘
- **那么** 系统将该表盘 status 设置为 approved

#### 场景:审核拒绝
- **当** 管理员拒绝某表盘
- **那么** 系统将该表盘 status 设置为 rejected
