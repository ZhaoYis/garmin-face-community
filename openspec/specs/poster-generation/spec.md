## 新增需求

### 需求:系统必须基于运动数据生成海报
用户必须能够选择运动记录和模板，生成个性化海报。

#### 场景:海报生成流程
- **当** 用户选择运动记录和模板后点击"生成海报"
- **那么** 系统使用 Canvas 渲染海报，展示预览并提供下载选项

#### 场景:数据自动填充
- **当** 用户选择一条运动记录
- **那么** 系统自动提取并格式化数据：比赛名称、完赛时间、平均配速、距离、心率、海拔

### 需求:海报必须支持高分辨率导出
系统必须支持导出适合社交媒体分享的高分辨率图片。

#### 场景:高分辨率导出
- **当** 用户下载海报
- **那么** 系统生成 2x 分辨率的 PNG 图片（1500×2668px）

### 需求:系统必须支持海报路线渲染
对于包含 GPS 数据的运动记录，海报必须展示路线图。

#### 场景:路线渲染
- **当** 运动记录包含 polyline 数据
- **那么** 系统在海报中渲染运动路线图

### 需求:用户必须能够保存海报
系统必须将生成的海报保存到用户账户。

#### 场景:海报保存
- **当** 用户确认保存海报
- **那么** 系统将海报图片上传到存储服务，并在数据库创建记录

### 需求:用户必须能够查看和管理海报
用户必须能够查看自己创建的所有海报。

#### 场景:海报列表
- **当** 用户访问"我的海报"页面
- **那么** 系统展示用户创建的所有海报缩略图

#### 场景:海报删除
- **当** 用户删除某张海报
- **那么** 系统从数据库和存储服务中删除该海报

### 需求:系统必须记录海报分享数据
当用户分享海报时，系统必须记录分享行为。

#### 场景:分享统计
- **当** 用户分享海报到社交平台
- **那么** 系统增加该海报的 shareCount 计数
